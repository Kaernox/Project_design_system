Components

More than 18 components to create amazing interfaces.

The Akkurate Design System is a series of individual AngularJS directives, that are designed to help developpers make websites faster. These designs are systemic and logical, as they all follow the same universal principles.
The component library gives developers a collection of re-usable HTML, JavaScript and SCSS partials for building their products. Check out the source code in the docs folder to see how we made these components work.
Here are the main components we have to offer.

Alert

The akk-select tag here is used to change the type of the alert so you can see the diffrent options it has to offer, another example it's of usage is available below.

Card

Here we have our akk-checkbox directive, it takes three values, the label, the model object and the property to take from that model object. The property will contain true if akk-checkbox is checked, or false.

Checkbox

Here we have our akk-checkbox directive, it takes three values, the label, the model object and the property to take from that model object. The property will contain true if akk-checkbox is checked, or false.

CheckboxList

Here we have our akk-checkbox-list directive, it's usefull when you have a list of options that you want to present to your users as a checkbox list, it taks the options to show the user and the model that will contain the selected items. 

{{ view.checkboxList | json }}

Color picker

The akk-colorpicker directive is an improvement of the angular colorpicker directive that provide a visual view of colors, it takes the model which will contain the color chosen by the user in a 6 digit hex format.

Datagrid

The akk-datagrid takes an array of objects and an array of columns names and displays it, the datagrid is sortable using any column, just click on the column name to sort, clicking multiple times will toggle the sort type between ascending and descending. 

{{view.datagrid.selectedItems | json }}

DatePicker

{{ view.date | json }}

The akk-datepicker allows you to pick a date and store it as a javascript date object in the value provided in the model property.

Input

The akk-input directive allows your users to type some text and stores it in the model property.

{{ view.inputValue ? view.inputValue : 'Start typing to see the result' }}

Loader

Youpi, it's loaded :)

Here we have our akk-loading, the property loading is binded with the checkbox so you can test this directive by indicating wether there is loading or not. To use it in your project, simply add the akk-loading directive as an attribute to any html element that you want to add a loading status to, and then assign a value to loading attribute, that value should be true when your content is loading and then changed to false when you content has finished loading.

Multiselect

The akk-multiselect directive takes a list of object (in the items property) and the field to be shown from those objects in the modal that will appear, selected objects will be removed from the items array and added to the selected array. 

{{ view.multiselect | json }}

Radio

The akk-radio is a set of radio buttons where options are specified in the options property as objects and the selected option will be in the model.
In property, you specify a field to be selected, model will contain the field vaue of the selected object, if no property has been specified, the model will contain the entire selected object, this will give you freedom to use primitives types as well as objects.
The akk-radio directive requires a all objects in options to have a property name so it can use it to display radio buttons. 

{{ view.radio | json }}

Select

The akk-select directive provides a modal with options to select from. Options are specified in the options property and the selected item will be stored in the model property. 

{{ view.select | json }}

SelectAndSearch

The akk-selectandsearch directive accepts an array of objects as input as well as how you want to display every object ("fields" attribute) in the modal that pops off, the object passed to the model attribute will contain the id of the selected object, the full list of objects is displayed using the akk-datagrid.
To start Selecting items, click on the 3 dots on the right. 

{{ view.selectAndSearch.selectedId | json }}

Switch

The akk-switch directive is simply a checkbox with a lot of verification to position the label correctly, you have three options for the button's position, you can edit it by setting the alignment property to right(default value), center or left, the example shows the three possible positions in order.
Size can also be increased by adding size="lg" to the directive. 

{{ view.switch.value | json }}

Textarea

The akk-textarea directive allows your users to type some text and stores it in the model property, you can increase the length of the text area as you like by dragging the bottom-right corner up and down.

{{ view.textareaValue ? view.textareaValue : 'Start typing to see the result' }}

Tree

The akk-tree directive takes a list of items with children (at any depth) and diplays them in a tree like format, the selected items of the tree will be stored within model.
